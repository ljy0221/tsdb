# Phase 0 벤치마크 결과 — 2026-04-09

## 환경
- OS: Linux (WSL2, 6.6.87.2-microsoft-standard)
- JVM: OpenJDK 22.0.2 (Temurin-22.0.2+9)
- JMH: 1.37
- Warmup: 3 iterations × 1s / Measurement: 5 iterations × 2s / Fork: 1
- 배열 크기: 1,000,000 elements

## 결과 (Throughput, ops/ms — 높을수록 좋음)

| Benchmark              | Score (ops/ms) | Error (±) |
|------------------------|---------------|-----------|
| heapRead               | 4.220         | 0.445     |
| **offHeapRead**        | **4.263**     | 0.414     |
| heapWrite              | 3.723         | 0.331     |
| **offHeapWrite**       | **3.653**     | 0.510     |

## 해석

- **읽기**: 오프힙(4.263) ≈ 인힙(4.220). 오프힙이 약 **1% 우세** (오차 범위 내).
- **쓰기**: 인힙(3.723) ≈ 오프힙(3.653). 인힙이 약 **2% 우세** (오차 범위 내).
- 전체적으로 **성능 차이 없음** — FFM API의 오버헤드가 사실상 0임을 확인.

## 결론

> `sun.misc.Unsafe` 없이 Java 22 FFM API만으로 인힙 배열과 동등한 성능 달성.
> Zero-GC 이점을 추가 성능 비용 없이 얻을 수 있다.
