# /project:blog-draft

오늘 구현한 내용을 바탕으로 Velog 포스팅 초안을 생성한다.

## 실행 순서

1. 오늘 변경된 파일 확인: `git diff --name-only HEAD~1`
2. 변경 내용 분석
3. docs/TIL.md 오늘 항목 확인
4. 최신 벤치마크 결과 확인
5. 포스팅 초안 생성

## 포스팅 스타일 (J의 Velog 스타일 기준)
- 독자: Java 백엔드 개발자, 시스템 프로그래밍 관심자
- 톤: 기술적으로 정확하되 읽기 쉽게. 왜 이렇게 했는지 이유 중심.
- 구성: 문제 정의 → 핵심 개념 → 구현 → 측정 결과 → 결론
- 코드 블록: 핵심만. 전체 코드는 GitHub 링크로 대체.
- 벤치마크 결과는 표로 정리.

## 출력 형식
```markdown
---
title: {제목}
tags: [Java, FFM API, TSDB, 오픈소스]
---

{본문 초안}
```
