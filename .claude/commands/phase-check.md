# /project:phase-check

현재 개발 Phase 상태를 점검하고 다음 액션을 제안한다.

## 실행 순서

1. CLAUDE.md의 Phase 체크리스트 읽기
2. `docs/benchmarks/` 최신 파일 확인
3. `docs/TIL.md` 최근 항목 확인
4. `git log --oneline -10` 최근 커밋 확인
5. 현재 상태 요약:
   - 완료된 태스크
   - 미완료 태스크
   - 다음으로 할 것 (1개만 추천)
   - 블로그 포스팅 가능한 시점인지 판단

## 출력 형식
```
## 현재 상태: Phase {N} — {Phase 이름}

### 완료
- [x] ...

### 미완료
- [ ] ...

### 오늘 할 것 (추천 1개)
> ...

### 블로그 가능 여부
> ...
```
